## §1 设计令牌（Design Tokens，第一步，一切的地基）

### 网格系统（贯穿全站，元素吸附不漂移）
```
页宽 max:      ~1440px
左导航列:      ~280–300px（sticky，博客式左栏）
右内容区:      ~1120px；内部 8 列 × 126px + 16px gutter（x.com 实测）
正文阅读宽:    640–760px（中文 30–40 字/行）——正文收窄于内容区
图版/数据/目录插画: 可用满内容区宽（877–1120）
```
承接 [封面网格对齐 memory]：所有区块元素吸附此网格，同侧共轴，不各自微调。首屏/二屏/三屏是全幅展示屏，可突破栏宽做居中大图，但仍落在页宽内。


### 颜色（软深色 + X 的不透明度阶梯 + 我们的强调）
```
--bg:            #08090A;   /* 画布底（unmoth 软近黑，非纯黑）*/
--surface-1:     #0d0d0d;   /* 抬升面/卡片 */
--surface-2:     #141414;   /* 更高抬升 */
--dot-texture:   #121314;   /* 极淡制图点阵 radial-gradient(#121314 .6px, transparent 1.4px) */
--hairline:      rgba(255,255,255,0.10);  /* 发丝线：全站唯一主分隔件 */
--ink:           #E8E8E6;   /* 正文主色（暖白非纯白，中文长读）*/
--ink-60:        rgba(255,255,255,0.60);  /* 次要文字/弱化标题 */
--ink-30:        rgba(255,255,255,0.30);  /* 占位/更弱 */
--ink-10:        rgba(255,255,255,0.10);  /* 极弱/等于发丝线 */
--accent:        #4337FF;   /* 品牌靛蓝，唯一强调 */
```
层级律：**靠亮度/不透明度分层（100→60→30→10），不靠颜色、少靠字重**（承接 §0 中文字重按字号反向调：大标题细、正文实）。

### 字体（三声部）
```
--font-serif: 'Noto Serif SC', 'Source Han Serif SC', serif;      /* 暖文学：标题/大句陈述 */
--font-sans:  'Noto Sans SC', 'PingFang SC', 'Hanken Grotesk', sans-serif; /* 中性：正文/UI（拉丁走 Hanken）*/
--font-mono:  'JetBrains Mono', ui-monospace, monospace;          /* 冷等宽：编号/图注/代码/标签 */
```
- **中文标题用思源宋**（暖文学声部，对应 unmoth 的 Faculty Glyphic 张力）。
- **中文正文用思源黑/苹方**，拉丁混排走 Hanken。
- **等宽只用于**：001 编号、mono 分节标签、图注、代码、规格读数。**中文正文绝不用 mono，中文绝不加负字距**（负字距只给拉丁+数字）。
- index.html 补加载 Noto Serif SC + Noto Sans SC。

### 字号阶梯（中文放大版，承接 animations.dev 配方：正文一个字号一个行高）
```
标题 h1: 44–52px 思源宋 / lh 1.2 / 拉丁数字可 -0.02em
标题 h2: 28–30px 思源宋 / lh 1.3
标题 h3: 20–22px 思源宋或黑 / lh 1.4
正文:    16px 思源黑 / lh 1.8（深底中文放大行高）/ #E8E8E6
mono 标签/编号/图注: 12–13px JetBrains Mono / 大写 / +0.02em
```

## §2 骨架组件（两站收敛，本次新增的「技术文档」母题）

1. **编号分节头**：每章/节顶一条发丝线，线上左 mono 标签（如 `设计工程 / ENGINEERING`）+ 右 mono 编号（`001`）。账本/规范式分节。
2. **同尺寸双色区块头**：两行同字号，第一行满亮当「标签」、第二行 60% 灰当「真标题」——只用亮度分主次（x.com 招牌）。
3. **双色词强调**：一句里关键词 `--ink` 亮、次要词 `--ink-60` 灰交替，不加粗不变色造阅读节奏（unmoth 招牌）。
4. **label→value 规格行**：`标签 → 值` + 细分隔线，用于原则速览/要点清单/对照。
5. **Bento 错落数据卡**：卡顶不对齐、高度参差、`--surface-1` 底 ~16px 圆角、大数字 count-up + 灰注解（x.com）；用于方法论指标/成效。
6. **单色蓝图线稿**：罗盘/雷达同心圆/等距线框/放射网络，虚线 + 方圆菱节点，`--ink` 白发丝线画在 `--bg` 上。作概念图语言（流程/关系/框架）。draw-on 描边入场。
7. **巨型近隐形 footer 字标**：超大「vibe designing playbook」用近 `--bg` 的灰渲染（几乎看不见）+ 虚线制图框 + 比例标注（unmoth 招牌）。
8. **克制按钮**：直角 SVG 描边、透明底、mono 大写、小尺寸；主 CTA 四角带蓝图角标 tick。

## §2.7 章节插画语言（学 polar.sh，做原创）

> 红线：**学视觉语言 + 动效机制，做我方原创插画**（代表我们章节），**不照抄 Polar 源码/资产**（Polar 开源仅供学习语言，源码在 scratchpad/src 仅参考手法）。

**家族一致性常量**（保多张图出自同一套）：
- 载体：静态屏 **SVG 单帧**即可 100% 还原观感；要动效循环则 **canvas 2D + dt-based rAF + IntersectionObserver 门控**（滚动进场才播，离开停）。
- **触达半径 = 0.32 × 画框**（体量一致、四周留白 ~36%，负空间为主）；一律**中心构图 + 放射对称**，锚 (size/2, size/2)。
- 线：纯白 `#fff` stroke、线宽 **1.5–2px**、平头端点(butt)、`fill:none`（皆空心环）、**无 glow/shadow**。
- **「幽灵轨道 + 高亮」二层**（最 polar 的招）：暗灰 `#3c3c3c` 画全貌常显 + 亮白只描/填一段做动画。任何描边动画都套这招。
- 卡底 `#101011` / hover `#161618` / 页底 `#090909`（与 §1 深色令牌统一，可微调对齐 #08090A）。

**动效工具箱**（精确值直接用）：
- 三缓动：描边/流动 `easeInOutCubic`、填充 `easeOutCubic`、旋转 线性。
- 旋转 **0.15–0.25 rad/s**（务必慢，一圈 25–42s）；描边画出 **0.8–2.8s**；错峰 **0.12–0.25s**；循环 **4–10s**；dt-based 帧率无关。

**概念 → 几何（我方章节原创方案，从图元库重组，不新发明）**：
- 引言 → 简洁开场（单同心环 / 罗盘准星）
- 设计工程/结构 → 同心环由内向外逐层描边（ConcentricDraw 式，"精密分层搭建"）或同心辐条带（GaugeSweep）
- 动态交互 → 箭头阵列跟随光标（VectorField 式，可交互最贴切）或流动循环（CycleArrow）
- 自我进化 → 自相似递归栈每层×0.62 旋转（OrbitingSpheres 式，"迭代生长"）或螺旋（polar 没有、同语言里很自然，我们补）
- 结语/词典 → 图元库再挑 2–4 个重组
- 图元库：环/嵌套环/外切环/绕行点/放射辐条/蛇形/箭头/发散条/明暗二层

**一图两用（统一系统）**：每章一张**签名插画**，小尺寸用在三屏目录卡、大尺寸+动画用在该章**章扉页**（§2.5）。全书插画同出一套语言。

**卡片承载**（三屏目录 = x.com feature grid ∩ polar Features）：grid 1/3 列 gap16；卡**直角无圆角**、`#101011` 底 hover `#161618`、内距 32；竖排 **标题(思源宋 30px 白) → 2px×40px 小分隔条 → 描述(20px 灰) → 底部 aspect-square 插画(mt-auto)**。

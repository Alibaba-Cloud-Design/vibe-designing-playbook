# Agent 调度看板 spec.md

## L1 定位与意图

- 一句话定义：用于查看 Agent 任务实时状态、失败原因和恢复入口的调度看板。
- 目标用户：运维、开发、平台管理员。
- 非目标：不在本页编辑 Agent 配置，不承载完整日志检索。

## L2 信息架构

- 顶部总览区：任务总数、运行中、失败、平均耗时。
- 主任务列表：任务名称、状态、触发人、耗时、最近更新时间、操作。
- 状态筛选区：全部、排队、运行中、失败、完成。
- 次级信息区：队列压力、失败趋势、最近失败原因。

## L3 核心链路

- 排队 → 运行中 → 完成。
- 排队 → 取消。
- 运行中 → 失败 → 查看原因 / 重试 / 忽略。
- 运行中 → 超时 → 重试 / 取消。

## L4 组件功能细节

- 状态 Badge：排队、运行中、完成、失败、超时。
- 任务表格：支持筛选、刷新、查看日志、重试。
- 趋势图：展示最近 24 小时任务成功和失败趋势。

## L5 边界条件

- 空态：说明当前没有任务，提供刷新入口。
- 加载态：短等待不闪烁，长等待展示骨架屏或 Loader。
- 错误态：接口失败时展示原因和重试入口。
- 权限不足：操作置灰并说明需要的权限。

## L6 验收标准

- 给定一个失败任务，用户能看到失败原因并触发重试。
- 给定无任务数据，页面展示空态，不出现白屏。
- 给定无权限用户，危险操作不可执行，并有原因说明。
